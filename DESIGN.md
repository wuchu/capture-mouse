# CaptureMouse - 键鼠共享方案设计文档

## 项目概述

一套创新的鼠标键盘共享方案，让 Windows 可以临时捕获并控制 macOS 设备。不同于传统方案，本方案基于 macOS 原生 Screen Sharing (VNC) 协议，macOS 端无需安装任何软件。

## 核心原理

```
┌─────────────────┐                      ┌─────────────────┐
│   Windows PC    │ ◄───── 键鼠设备 ────► │   USB HID       │
│  (控制端软件)    │                      │  (物理设备)      │
│                 │                      └─────────────────┘
│  ┌───────────┐  │
│  │ 输入捕获   │  │
│  │ (Raw Input)│  │
│  └─────┬─────┘  │
│        ▼        │
│  ┌───────────┐  │      VNC协议        ┌─────────────────┐
│  │ VNC客户端  │◄─┼────────────────────►│   macOS         │
│  │ (键鼠事件) │  │  (只发事件不收画面)  │  Screen Sharing │
│  └───────────┘  │                      │  (原生服务)      │
└─────────────────┘                      └─────────────────┘
```

## 技术架构

### 1. 输入捕获层 (InputCapture)
- Windows Raw Input API 捕获全局键鼠事件
- 事件队列管理
- 设备状态追踪

### 2. VNC 客户端层 (VncClient)
- RFB 3.8 协议实现
- 仅实现键鼠事件发送
- 不接收画面数据（节省带宽）

### 3. 控制管理层 (ControlManager)
- 状态机管理（本地/远程模式）
- 全局快捷键监听
- 系统托盘交互

### 4. 配置层 (Config)
- IP 地址配置
- 快捷键配置
- 持久化存储

## 技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 开发语言 | C# .NET 9 | 最新版本，Windows 11 专属 |
| UI 框架 | System.Windows.Forms | 轻量级，系统托盘支持 |
| 输入捕获 | Raw Input API | 全局设备捕获 |
| 网络协议 | TCP + RFB 3.8 | VNC 标准协议 |
| 快捷键 | RegisterHotKey | Windows 原生 API |

## 快捷键设计

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + Alt + M` | 切换控制权 (本地 ↔ 远程) |
| `Ctrl + Alt + Q` | 退出程序 |

## 状态指示

系统托盘图标颜色：
- ⚪ 灰色 = 未连接
- 🔵 蓝色 = 已连接，本地模式
- 🟢 绿色 = 已连接，远程模式

## macOS 端配置

1. 系统设置 → 通用 → 共享 → 屏幕共享 → 开启
2. 设置访问密码
3. 获取 IP 地址（系统设置 → 网络）

- 使用 VNC 密码认证

- 建议在同一局域网内使用
